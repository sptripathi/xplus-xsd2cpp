Why W3C XML-Schema?

How do I constrain a well-formed XML document to a schema?

Answer:
The XML file can have a schema specification for it.
There are at least two standard ways of providing schema for XML documents:
  • DTD - Document Type Definition
  • W3C XML-Schema
Any such schema serves the purpose of validating the XML document against it, among other things. A XML document that validates against it's schema is called valid XML document.

The W3C XML-Schema is considered to be more powerful in expressing the schema of the XML-document, than DTD. The W3C XML-Schema specification is also the more popular one, than DTD or other similar schema specifications.
To know more about XML-Schema, read W3C specifications on it. Following are a few related useful links:
  • XML Schema Part 0: Primer Second Edition
  • XML Schema Part 1: Structures Second Edition
  • XML Schema Part 2: Datatypes Second Edition
Henceforth, across different sections of XmlPlus documentation, the term XML-Schema means W3C XML-Schema.

Why Data-Binding?

Given that we have a XML-Schema for my XML file, how do we write and read only the valid XML files ?
If we use DOM API to write the XML files then it may take many iterations before we write the valid XML file. Where is the solution?

Answer:
The solution is to use data-bindings for the XML-Schema.
The data-binding implementations typically provide following utilities:
  • The data-binding structures for XML-Schema eg. C++ classes for XML-Schema components
  • Parser to read valid XML documents into data-binding structures
  • Writer to write the data-bindings to a xml file
There are many data-binding solutions available across several platforms/programming-languages.
